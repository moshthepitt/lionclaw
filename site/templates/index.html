Local agent home

Your agent should work from your side of the fence.

AI agents will read your files, remember your work, use your keys, message people, and run jobs.

That should not all live inside someone else's platform.

LionClaw gives real agent CLIs a local home in your project: sessions, state, channels, jobs, and audit.

Use Codex. Use OpenCode. Use whatever comes next. Keep the home.

Start

Start where your work already is.

No new platform to move into. No dashboard as the source of truth.

Run doctor. Fix what it reports. Then run the agent. 

git clone https://github.com/moshthepitt/lionclaw.git
cd lionclaw
cargo build --release
export PATH="$PWD/target/release:$PATH"

cd /path/to/your/project
lionclaw project init
lionclaw doctor
lionclaw run

Why LionClaw

An agent is not just chat.

It reads files. It writes code. It remembers decisions. It can use credentials. It can keep working after you leave.

That power should not need a landlord.

LionClaw gives it a local home.

Your work

Your record

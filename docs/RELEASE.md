# Release Process

LionClaw releases are a two-stage GitHub Actions flow.

## Steps

1. Run the `Create Release PR` workflow from the Actions page and choose a SemVer bump level: `major`, `minor`, or `patch`.
2. Review and merge the generated `release/vX.Y.Z` pull request into `main`.
3. Wait for the `release` workflow to build the binaries, push tag `vX.Y.Z`, and publish the GitHub Release.

## What gets published

Each GitHub Release includes:

- `lionclaw-vX.Y.Z-linux-x86_64.tar.gz`
- `SHA256SUMS`

The tarball contains:

- `lionclaw`
- `lionclawd`
- `LICENSE`

## Notes

- In GitHub repository settings, Actions must allow `GITHUB_TOKEN` write access and allow GitHub Actions to create pull requests.
- The generated release PR is labeled `release`.
- GitHub autogenerated release notes are enabled for the published release.
- The `release` label is excluded from autogenerated notes through `.github/release.yml`, so the generated release PR itself does not appear in the changelog.
